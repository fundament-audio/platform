# PHILOSOPHY

Why FUNDAMENT exists.

This document defines the principles behind every FUNDAMENT build.

Individual builds implement these principles. They do not redefine them.

---

# Engineering first

FUNDAMENT is an engineering project.

The goal is not to build "the best speaker."

The goal is to build loudspeakers whose engineering is transparent, measurable, understandable and reproducible.

Good engineering should survive hype, trends and marketing.

Measurements matter.

Documentation matters.

Revision history matters.

---

# Open by default

Everything that can reasonably be open should be open.

That includes:

* enclosure drawings
* CAD
* cut lists
* DSP presets
* protection filters
* measurements
* design notes
* bills of materials
* revision history

Knowledge should not disappear because a forum thread dies or a website goes offline.

---

# Reference, not perfection

Every FUNDAMENT build has one goal:

Create a documented reference implementation.

The reference build is measured.

The methodology is published.

The data is published.

The conclusions are published.

That does not mean every copy built in a garage automatically performs identically.

Wood varies.

Assembly varies.

Drivers have tolerances.

People make mistakes.

Our job is not to pretend those things don't exist.

Our job is to document them honestly.

---

# Validate. Don't guess.

Opinions are useful.

Measurements decide.

Whenever practical, engineering decisions should be backed by:

* simulation
* measurements
* listening
* repeatability

If new evidence proves something better, the project should improve.

Being wrong is perfectly acceptable.

Refusing to update isn't.

---

# Modularity is a feature

Drivers wear out.

Amplifiers fail.

DSP platforms become obsolete.

Suppliers discontinue products.

That should never mean throwing away an otherwise excellent loudspeaker.

A FUNDAMENT build should be designed so individual parts can be replaced, repaired or upgraded over time.

The enclosure is a platform.

The rest can evolve.

---

# Convenience should never become lock-in

Not everyone wants to build loudspeakers.

Some people want plans.

Some want a flatpack.

Some want assembled cabinets.

Some want a complete ready-to-play system.

Those are all valid.

Services exist to reduce effort, not to hide knowledge.

The engineering always stays open.

The documentation always stays public.

Buying convenience should never be the only way to access good engineering.

---

# Recommend the right solution

FUNDAMENT does not believe in one "best" loudspeaker.

Every design is a compromise.

Every driver is a compromise.

Every amplifier is a compromise.

Every cabinet is a compromise.

The interesting question is never:

> Which one is best?

It's:

> Which one is best for this person, in this room, for this use case?

That's why FUNDAMENT documents tradeoffs, not winners.

---

# Build paths, not product variants

A FUNDAMENT build may support multiple validated ways of building the same loudspeaker.

Different drivers.

Different DSP paths.

Different amplifiers.

Different cabinet materials.

Different finishes.

Different levels of DIY.

Those are not random options.

They are documented build paths with different goals, different compromises and different audiences.

If a combination has not been validated, it should not be presented as an official recommendation.

---

# Repairability matters

We build things that should still make sense ten or twenty years from now.

If a driver fails, replace the driver.

If an amplifier dies, replace the amplifier.

If a DSP platform disappears, migrate to another documented solution.

If a better driver becomes available, validate it and publish a new build path.

Ownership should not end because one component became unavailable.

---

# Documentation is part of the product

A loudspeaker without documentation is difficult to reproduce.

A measurement without methodology is difficult to trust.

A design without revision history is difficult to improve.

Documentation is not something you write afterwards.

It is part of the engineering.

---

# Community over personalities

FUNDAMENT should become more useful over time.

That only happens if knowledge is shared.

Build it.

Measure it.

Challenge it.

Improve it.

Document it.

Repeat.

The goal is that someone who discovers FUNDAMENT ten years from now has more knowledge available than we do today.

Not because one person knew everything.

Because many people contributed.

---

# Long-term thinking

The internet is full of excellent loudspeaker projects that slowly disappeared.

Images vanished.

Links broke.

DSP files were lost.

Measurements became impossible to find.

FUNDAMENT exists because good engineering deserves a permanent home.

The goal isn't simply to build loudspeakers.

The goal is to preserve engineering knowledge in a form that can be built, measured, repaired, improved and trusted for decades.

If we manage that, we've built something much bigger than a speaker.
